# DealClaw 社媒效果预览界面设计方案

## 1. 方案概述

### 1.1 核心目标
解决国内用户无法直接访问 LinkedIn/Facebook 等海外社媒平台的问题，通过 DealClaw 后台实现社媒内容的镜像展示和互动管理，让用户无需翻墙即可：
- 实时查看 AI 在社媒上发布的内容
- 查看和管理评论、私信等互动内容
- 预览内容发布效果
- 统一回复和管理社媒互动

### 1.2 方案定位
- **功能定位**：社媒渠道的内容镜像与互动管理中心
- **用户价值**：零门槛查看海外社媒运营效果，提升运营效率
- **技术定位**：官方 API 为主，结合智能同步策略

| 模块编号 | 模块名称 | 核心功能 | 优先级 |

### 3.2 社媒内容镜像展示系统 (SM-01)

#### 3.2.1 功能说明

实时同步 LinkedIn/Facebook 上的内容到 DealClaw 后台，提供与原生平台一致的浏览体验。

#### 3.2.2 同步内容类型

| 内容类型 | LinkedIn | Facebook | 同步策略 |
|---------|----------|----------|---------|
| 帖子 (Post) | ✅ | ✅ | 全量同步，15分钟轮询 |
| 图片 (Image) | ✅ | ✅ | 增量同步，实时Webhook |
| 视频 (Video) | ✅ | ✅ | 元数据同步，缩略图缓存 |
| 链接卡片 (Link) | ✅ | ✅ | 全量同步 |
| 评论 (Comment) | ✅ | ✅ | 增量同步，实时Webhook |
| 点赞数据 (Likes) | ✅ | ✅ | 每小时聚合同步 |
| 分享数据 (Shares) | ✅ | ✅ | 每小时聚合同步 |
| 私信 (Message) | ✅ | ✅ | 实时推送 |

#### 3.2.4 同步频率配置

| 数据类型 | 同步频率 | 最大延迟 | 数据保留策略 |
|---------|---------|---------|------------|
| 帖子内容 | 15分钟 | 15分钟 | 永久保留 |
| 评论内容 | 实时(Webhook) | 5秒 | 永久保留 |
| 私信内容 | 实时(Webhook) | 3秒 | 永久保留 |
| 点赞/分享数 | 1小时 | 1小时 | 保留90天 |
| 媒体文件 | 实时 | 30秒 | 缓存30天 |
| 账号信息 | 24小时 | 24小时 | 永久保留 |

**API调用架构**
```typescript
// LinkedIn API 客户端
class LinkedInAPIClient {
  private accessToken: string;
  private baseUrl = 'https://api.linkedin.com/v2';
  
  // 获取企业主页帖子
  async getOrganizationPosts(organizationId: string, pagination?: PaginationParams) {
    const response = await fetch(
      `${this.baseUrl}/posts?author=urn:li:organization:${organizationId}`,
      {
        headers: {
          'Authorization': `Bearer ${this.accessToken}`,
          'X-Restli-Protocol-Version': '2.0.0'
        }
      }
    );
    return response.json();
  }
  
  // 获取帖子评论
  async getPostComments(postId: string) {
    const response = await fetch(
      `${this.baseUrl}/socialActions/${postId}/comments`,
      {
        headers: { 'Authorization': `Bearer ${this.accessToken}` }
      }
    );
    return response.json();
  }
  
  // 发布评论回复
  async createCommentReply(commentId: string, message: string) {
    const response = await fetch(
      `${this.baseUrl}/socialActions/${commentId}/comments`,
      {
        method: 'POST',
        headers: {
          'Authorization': `Bearer ${this.accessToken}`,
          'Content-Type': 'application/json'
        },
        body: JSON.stringify({
          actor: this.getCurrentActor(),
          message: { text: message }
        })
      }
    );
    return response.json();
  }
}
```

#### 4.2.2 Webhook 事件处理

```typescript
// Webhook 事件处理器
interface WebhookEvent {
  platform: 'linkedin' | 'facebook';
  eventType: 'comment' | 'message' | 'post' | 'reaction';
  timestamp: string;
  payload: any;
}

class SocialMediaWebhookHandler {
  async handleEvent(event: WebhookEvent) {
    // 1. 去重检查
    if (await this.isDuplicate(event)) {
      return { status: 'ignored', reason: 'duplicate' };
    }
    
    // 2. 根据事件类型分发处理
    switch (event.eventType) {
      case 'comment':
        await this.handleCommentEvent(event);
        break;
      case 'message':
        await this.handleMessageEvent(event);
        break;
      case 'reaction':
        await this.handleReactionEvent(event);
        break;
    }
    
    // 3. 发送实时通知到前端
    await this.sendRealtimeNotification(event);
    
    return { status: 'processed' };
  }
  
  private async handleCommentEvent(event: WebhookEvent) {
    // 保存评论到数据库
    const comment = await this.saveComment(event.payload);
    
    // AI意图识别
    const intent = await this.aiService.classifyIntent(comment.text);
    
    // 标记高意向评论
    if (intent.score > 0.7) {
      await this.markAsHighIntent(comment.id);
    }
    
    // 生成回复建议
    const replySuggestion = await this.aiService.generateReply(comment.text, intent);
    await this.saveReplySuggestion(comment.id, replySuggestion);
  }
}
```

// 缓存管理器
class SocialMediaCache {
  private redis: Redis;
  
  // 帖子列表缓存 (TTL: 5分钟)
  async getCachedPosts(accountId: string, page: number) {
    const cacheKey = `posts:${accountId}:${page}`;
    let posts = await this.redis.get(cacheKey);
    
    if (!posts) {
      posts = await this.fetchPostsFromDB(accountId, page);
      await this.redis.setex(cacheKey, 300, JSON.stringify(posts));
    }
    
    return JSON.parse(posts);
  }
  
  // 未读消息数缓存 (TTL: 1分钟)
  async getUnreadCount(accountId: string) {
    const cacheKey = `unread:${accountId}`;
    let count = await this.redis.get(cacheKey);
    
    if (!count) {
      count = await this.calculateUnreadCount(accountId);
      await this.redis.setex(cacheKey, 60, count.toString());
    }
    
    return parseInt(count);
  }
  
  // 缓存失效策略
  async invalidateCache(event: CacheInvalidationEvent) {
    switch (event.type) {
      case 'new_post':
        await this.redis.del(`posts:${event.accountId}:*`);
        break;
      case 'new_comment':
        await this.redis.del(`comments:${event.postId}`);
        await this.redis.del(`unread:${event.accountId}`);
        break;
      case 'new_message':
        await this.redis.del(`messages:${event.conversationId}`);
        await this.redis.del(`unread:${event.accountId}`);
        break;
    }
  }
}

// WebSocket 连接管理
class SocialMediaRealtimeService {
  private ws: WebSocket;
  private eventHandlers: Map<string, Function[]> = new Map();
  
  connect(accountId: string) {
    this.ws = new WebSocket(`wss://api.dealclaw.com/social-media/ws?account=${accountId}`);
    
    this.ws.onmessage = (event) => {
      const data = JSON.parse(event.data);
      this.handleEvent(data);
    };
  }
  
  private handleEvent(event: RealtimeEvent) {
    const handlers = this.eventHandlers.get(event.type) || [];
    handlers.forEach(handler => handler(event.payload));
  }
  
  onNewComment(callback: (comment: Comment) => void) {
    this.subscribe('new_comment', callback);
  }
  
  onNewMessage(callback: (message: Message) => void) {
    this.subscribe('new_message', callback);
  }
  
  private subscribe(eventType: string, callback: Function) {
    if (!this.eventHandlers.has(eventType)) {
      this.eventHandlers.set(eventType, []);
    }
    this.eventHandlers.get(eventType)!.push(callback);
  }
}

| 安全措施 | 实现方式 |
|---------|---------|
| API密钥加密 | AES-256-GCM加密存储 |
| 传输加密 | TLS 1.3 |
| 访问控制 | OAuth 2.0 + RBAC |
| 审计日志 | 记录所有社媒操作 |

### 7.2 隐私合规

| 合规项 | 措施 |
|-------|------|
| 数据最小化 | 仅同步必要的社媒数据 |
| 用户授权 | 明确获取社媒平台授权 |
| 数据删除 | 支持断开连接后删除数据 |
| 地区合规 | 遵守GDPR、CCPA等法规 |

### 7.3 平台规则遵守

| 平台 | 限制 | 防护措施 |
|-----|------|---------|
| LinkedIn | API调用频率限制 | 请求队列 + 指数退避 |
| Facebook | 消息发送限制 | 频率控制 + 内容审核 |
| 通用 | 自动化工具检测 | 模拟人工操作间隔 |

## 8. 实施建议

### 8.1 开发阶段规划

| 阶段 | 周期 | 核心功能 |
|-----|------|---------|
| 第一阶段 | 2周 | 账号绑定、基础内容同步、时间线展示 |
| 第二阶段 | 2周 | 评论管理、私信管理、AI回复建议 |
| 第三阶段 | 1周 | 内容发布预览、数据看板 |
| 第四阶段 | 1周 | 优化、测试、性能调优 |

### 8.2 关键成功指标

| 指标 | 目标值 | 测量方式 |
|-----|-------|---------|
| 数据同步延迟 | <5分钟 | 对比平台时间和同步时间 |
| 页面加载时间 | <2秒 | Lighthouse Performance |
| 消息推送延迟 | <3秒 | WebSocket事件测量 |
| 用户满意度 | >4.5/5 | 用户调研 |

### 8.3 风险与应对

| 风险 | 可能性 | 影响 | 应对措施 |
|-----|-------|------|---------|
| API变更 | 中 | 高 | 关注官方文档，预留适配层 |
| 授权失效 | 高 | 中 | 自动提醒重授权，保留历史数据 |
| 同步失败 | 中 | 中 | 失败重试机制，降级展示缓存数据 |
| 数据量大 | 中 | 中 | 分页加载，增量同步，数据归档 |
