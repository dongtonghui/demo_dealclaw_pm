## 🎯 一、产品定位：从「工具」到「数字员工团队」

### 1.1 定位转变

| 维度 | 旧思维（V2.2） | 新思维（V3.0） |
|-----|---------------|---------------|
| **产品形态** | 双引擎获客工具 | **对话式AI数字员工团队** |
| **用户角色** | 工具使用者 | **团队管理者** |
| **交互方式** | 配置界面操作 | **自然语言对话（唯一入口）** |
| **价值交付** | 功能使用权限 | **可成交的商机结果** |

### 1.2 一句话定位

> **DealClaw是首个为外贸B2B打造的「对话式AI数字员工团队」。**
>
> 用户通过自然语言与主管Agent对话，主管Agent理解需求、协调专业Agent团队（SEO/Email/社媒等）、整合多渠道能力，全程无需人工干预，交付可成交的商机。

## 🏗️ 二、架构设计：单一入口 + 职责分离 + 可扩展

### 2.1 三层架构模型

```
┌─────────────────────────────────────────┐
│              用户层                      │
│    外贸企业主 / 外贸业务员 / 运营人员     │
└─────────────────┬───────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│              对话层                      │
│         数字员工团队主管Agent             │
│    （唯一对话入口、任务协调、结果汇报）   │
└─────────────────┬───────────────────────┘
                  │
        ┌─────────┼─────────┐
        ▼         ▼         ▼
   ┌────────┐ ┌────────┐ ┌────────┐
   │SEO Agent│ │Email Agent│ │WhatsApp│
   │（内容） │ │（外联） │ │（即时） │
   └────────┘ └────────┘ └────────┘
                  │
                  ▼
┌─────────────────────────────────────────┐
│              能力层                      │
│    数据能力 / 内容能力 / 触达能力        │
│    风控能力 / 分析能力 / 安全合规        │
└─────────────────────────────────────────┘
```

### 2.2 核心设计原则

| 原则 | 说明 | 用户价值 | 技术价值 |
|-----|------|---------|---------|
| **单一对话入口** | 用户只与主管Agent对话，无需切换界面 | 降低90%学习成本 | 统一接口、简化交互 |
| **职责分离** | 主管Agent只做管理协调，子Agent做专业执行 | 明确责任边界 | 架构清晰、易于维护 |
| **可扩展架构** | 支持新增Agent角色，形成数字员工生态 | 持续获得新能力 | 微服务架构、插件化 |

## 💬 四、交互范式：对话即界面（CUI）

### 4.1 自然语言驱动一切

| 用户意图 | 传统SaaS操作 | DealClaw对话式 |
|---------|------------|---------------|
| **创建获客任务** | 填写10+字段表单 | "帮我找美国户外用品批发商" |
| **查看任务进度** | 进入仪表盘查看 | "我的任务进展如何？" |
| **获取新线索** | 导出报表筛选 | "今天有什么新线索？" |
| **优化邮件内容** | 进入编辑器修改 | "@Email Agent 优化这封邮件" |
| **查看获客效果** | 生成报表分析 | "这周获客效果如何？" |

### 4.2 富媒体卡片系统

对话中嵌入可交互的富媒体卡片，替代传统表单和仪表盘：

| 卡片类型 | 用途 | 交互能力 |
|---------|------|---------|
| **画像确认卡片** | 展示AI理解的目标客户画像 | 编辑、确认 |
| **获客方案卡片** | 展示各Agent策略组合 | 一键确认、调整配置 |
| **内容预览卡片** | 展示生成的SEO文章/邮件 | 编辑、预览、发布 |
| **线索摘要卡片** | 展示高意向线索信息 | 查看详情、生成回复、分配 |
| **数据看板卡片** | 展示核心获客指标 | 下钻查看、导出数据 |

## 🛠️ 五、技术架构：微服务 + 多Agent协作

### 5.1 系统架构四层模型

| 层级 | 职责 | 核心组件 | 技术选型 |
|-----|------|---------|---------|
| **接入层** | 流量入口、安全防护 | API Gateway、CDN、WAF | Kong/Nginx + Cloudflare |
| **服务层** | 业务逻辑、Agent协作 | 对话服务、Agent调度、业务支撑 | Node.js/NestJS + Temporal |
| **数据层** | 数据存储、缓存加速 | 主数据、文档、时序、缓存 | PostgreSQL + MongoDB + ClickHouse + Redis |
| **集成层** | 外部服务对接 | LLM、邮件、WhatsApp、站点 | Kimi/Claude + SendGrid + WhatsApp API |

### 5.2 Agent协作机制

```
主管Agent（协调者）
    │
    ├──► 意图识别 → 实体提取 → 上下文管理
    │
    ├──► 任务拆解（并行派发）
    │     ├── SEO Agent: 内容策略、关键词分析、文章生成
    │     ├── Email Agent: 客户筛选、邮件生成、发送策略
    │     └── WhatsApp Agent: 自动回复、线索捕获
    │
    ├──► 结果聚合 → 生成获客方案卡片
    │
    ├──► 执行监控 → 定时汇报
    │
    └──► 线索评分 → 高意向推送
```

### 5.3 关键技术指标

| 指标 | 目标值 | 说明 |
|-----|-------|------|
| **对话响应延迟** | < 3秒 | 用户输入到AI回复首字 |
| **内容生成速度** | > 50 tokens/秒 | 邮件/文章内容生成 |
| **数据查询响应** | < 500ms | 企业库搜索和筛选 |
| **系统可用性** | 99.9% | 月度停机时间<43分钟 |
| **并发用户数** | 10,000+ | 同时在线使用对话功能 |

## ✅ 八、关键设计决策记录

| 决策项 | 决策选择 | 决策理由 | 影响评估 |
|-------|---------|---------|---------|
| **MVP范围** | 双引擎（SEO+Email） | 与纯外联竞品形成差异化 | 复杂度增加，但价值主张更完整 |
| **主管Agent职责** | 只做协调，不做执行 | 架构清晰、责任明确 | 需设计清晰的Agent接口规范 |
| **SEO获客模式** | 三种模式并存 | 适配不同企业阶段（有站/无站/平台） | 技术实现复杂度增加 |
| **交互入口** | 自然语言唯一入口 | 零学习成本、提升使用频率 | 对意图识别精度要求高 |
| **邮件发送渠道** | 三种渠道可选（自有/代发/专业服务商） | 平衡成本与送达率 | 需维护多供应商对接 |
| **技术栈选型** | Node.js + PostgreSQL + Kubernetes | 团队熟悉、生态成熟 | 长期维护成本低 |

## 📝 十、总结

### 10.1 核心设计公式

```
DealClaw = 对话式AI数字员工团队
         = 单一对话入口
         + 主管Agent协调
         + 多子Agent协作
         + 双引擎获客
         + 可扩展架构
```

### 10.2 设计原则总结

| 层面 | 核心原则 | 关键举措 |
|-----|---------|---------|
| **用户视角** | 零学习成本 | 自然语言交互、富媒体卡片、主动引导 |
| **产品视角** | 结果导向 | 交付可成交商机、而非功能使用权限 |
| **技术视角** | 微服务架构 | 服务解耦、Agent独立、数据分层 |
| **商业视角** | 成本递减 | 双引擎协同、资产积累、长期价值 |

### 10.3 竞争壁垒总结

> **短期靠体验，中期靠架构，长期靠生态**

- **短期（现在）**：对话式交互体验领先、双引擎协同能力
- **中期（6-12月）**：企业知识库积累、多Agent协调算法优化
- **长期（12月+）**：完整外贸数字员工生态、数据飞轮效应
