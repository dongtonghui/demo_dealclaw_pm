## 二、核心设计思路

### 2.1 产品定位升级

**从V2.2到V3.0的定位转变：**

| 维度 | V2.2 | V3.0 |
|-----|------|------|
| 产品形态 | 双引擎获客工具 | **对话式AI数字员工团队** |
| 用户角色 | 工具使用者 | **团队管理者** |
| 交互方式 | 对话式（功能导向） | **对话式（员工协作导向）** |
| 核心差异 | 双引擎（数据+SEO） | **数字员工协作+双引擎+可扩展** |

**一句话定位：**
> DealClaw是首个为外贸B2B打造的「对话式AI数字员工团队」。用户通过自然语言与「主管Agent」对话，主管Agent理解需求、协调专业Agent团队、整合多渠道能力，全程无需人工干预，交付可成交的商机。

### 2.2 架构设计核心原则

#### 原则1：单一对话入口
- 用户只与「主管Agent」对话，无需切换界面
- 所有功能通过自然语言触发
- 富媒体卡片作为对话的延伸

#### 原则2：职责分离
| Agent类型 | 职责 | 执行方式 |
|----------|------|---------|
| 主管Agent | 理解、拆解、协调、汇报 | **不直接执行具体获客动作** |
| 子Agent | 专业领域执行 | SEO/Email/WhatsApp等具体任务 |

#### 原则3：可扩展架构
- 当前：主管Agent + SEO Agent + Email Agent（P0）
- P1：+ WhatsApp Agent + 社媒Agent
- P2/P3：+ 物流Agent + 竞对监控Agent + 选品报价Agent + 客服Agent

### 2.3 双引擎获客模型

```
数据外联引擎（即时见效）          SEO内容引擎（长期积累）
       ↓                                    ↓
主动触达潜在客户                    被动吸引搜索流量
       ↓                                    ↓
快速获取首批线索和反馈              积累内容资产和品牌权重
       ↓                                    ↓
验证市场和优化话术                  降低长期获客成本
       ↓                                    ↓
└───────────┬────────────────────────────┘
            ↓
     形成获客飞轮效应
     短期见效 + 长期降本
```

**成本对比（12个月周期）：**
- 竞品纯外联：恒定¥150/线索
- DealClaw综合：从¥200递减至¥100/线索

### 2.4 差异化价值（vs 核心竞品）

| 对比维度 | Workus/iSales | DealClaw |
|---------|--------------|----------|
| **交互方式** | 配置界面 | **自然语言对话** |
| **获客模式** | 纯主动外联 | **主动+被动双引擎** |
| **资产积累** | 无 | **独立站+内容库** |
| **长期成本** | 恒定 | **递减** |
| **扩展性** | 功能模块 | **数字员工角色** |

## 三、关键决策记录

### 3.1 MVP范围决策

**决策：MVP包含双引擎（SEO+Email），而非单引擎**
- 决策者：用户（产品经理）
- 理由：与Workus/iSales等纯外联竞品形成差异化
- 影响：MVP复杂度增加，但价值主张更完整

**MVP Agent清单：**
| Agent | 优先级 | 核心功能 |
|-------|-------|---------|
| 主管Agent | P0 | 统一入口、任务协调、结果汇报 |
| SEO Agent | P0 | 三种模式配置、内容生成、站点优化 |
| Email Agent | P0 | 客户筛选、个性化邮件、发送策略 |
| WhatsApp Agent | P1 | 即时通讯、线索捕获、自动回复 |

### 3.2 架构设计决策

**决策：主管Agent不直接执行具体任务**
- 职责：理解、拆解、协调、监控、汇报
- 执行：全部委托给子Agent
- 价值：架构清晰、易于扩展、责任明确

### 3.3 SEO模式决策

**三种模式并存，由用户/AI根据情况选择：**
1. **模式一（自引流）**：已有独立站，AI优化+内容运营
2. **模式二（AI建站）**⭐推荐：无独立站，AI一键建站+运营
3. **模式三（平台引流）**：借助平台流量，快速获取曝光

### 3.4 邮件发送渠道决策

**三种渠道可选：**
1. **自有邮箱**：OAuth/SMTP配置，免费但送达率中等
2. **平台代发**：高送达率(95%+)，按量付费
3. **专业服务商**：SendGrid/SES，企业级送达率(98%+)

## 五、核心交互流程

### 5.1 首次使用（Onboarding）
```
欢迎介绍 → 企业知识学习（对话/上传） → 生成画像卡片 
→ 画像确认 → 需求探询 → 获客策略建议 → 渠道配置引导 
→ 完成Onboarding
```

### 5.2 创建获客任务
```
用户输入目标 → 主管Agent理解（结合企业知识库） → 追问细节 
→ 生成画像卡片 → 确认画像 → 任务拆解（协调子Agent） 
→ 子Agent并行执行 → 汇总生成方案卡片 → 确认执行 
→ 执行监控 → 线索交付
```

### 5.3 日常交互模式
- **主动汇报**：主管Agent每日推送获客简报
- **被动查询**：用户询问，主管Agent实时响应
- **专业对话**：@指定Agent进行深度交互
