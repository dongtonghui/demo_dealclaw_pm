# DealClaw 评估指标体系设计

> 版本: V1.0  
> 更新日期: 2026-03-30  
> 设计目标: 建立Agent-native产品的专属评估体系

---

## 1. 核心设计原则

### 1.1 Agent产品 vs 传统产品评估

| 维度 | 传统产品评估 ❌ | Agent产品评估 ✅ |
|------|---------------|----------------|
| 成功标准 | 功能完整可用 | 任务独立完成率 |
| 用户行为 | 点击/停留/转化 | 对话轮数/干预率 |
| 质量指标 | Bug数/性能 | 意图准确率/边界触发率 |
| 长期价值 | 留存/付费 | 记忆质量/学习效率 |

### 1.2 Agent评估三问

1. **Agent能独立完成任务吗？** - 任务完成率、人工干预率
2. **Agent的决策用户信任吗？** - 用户接受率、置信度分布
3. **Agent在正确的时候停下来吗？** - 安全边界触发率

## 3. 核心指标详细定义

### 3.1 意图识别准确率

**定义**: Agent正确识别用户意图的比例

```
准确率 = 正确识别的意图数 / 总意图识别次数 × 100%
```

**测量方法**:
- 抽样标注: 每周随机抽取100条对话，人工标注正确意图
- 用户反馈: 用户对"我理解对吗？"的确认率
- 纠正率: 用户纠正意图理解的比例（反向指标）

**目标值与分级**:

| 等级 | 准确率范围 | 状态 |
|-----|-----------|------|
| 优秀 | >95% | 🟢 健康 |
| 良好 | 90%-95% | 🟡 监控 |
| 需优化 | 85%-90% | 🟠 警告 |
| 严重 | <85% | 🔴 告警 |

**细分维度**:

| 意图类型 | 目标准确率 | 说明 |
|---------|-----------|------|
| 创建任务 | >95% | 核心流程，必须准确 |
| 查询进度 | >90% | 相对简单 |
| 获取线索 | >92% | 高频操作 |
| 优化内容 | >85% | 较复杂，允许一定误差 |
| 系统操作 | >90% | 需明确 |

---

### 3.2 任务完成率

**定义**: Agent独立完成完整任务流程的比例

```
完成率 = 独立完成的任务数 / 总任务数 × 100%

独立完成的定义:
- 无需人工介入澄清意图
- 无需人工补充关键信息
- 无需人工处理异常
- 用户接受最终结果
```

**测量方法**:
- 任务日志分析
- 人工标注抽样（每任务10%）
- 用户反馈收集

**目标值与分级**:

| 等级 | 完成率范围 | 状态 |
|-----|-----------|------|
| 优秀 | >85% | 🟢 |
| 良好 | 75%-85% | 🟡 |
| 需优化 | 65%-75% | 🟠 |
| 严重 | <65% | 🔴 |

**任务类型细分**:

| 任务类型 | 目标完成率 | 难度系数 |
|---------|-----------|---------|
| 创建获客任务 | >80% | 高（多轮交互） |
| 查询任务进度 | >95% | 低 |
| 获取线索 | >90% | 中 |
| 优化邮件内容 | >75% | 高（主观性强） |
| 更新企业画像 | >85% | 中 |

---

### 3.3 人工干预率

**定义**: 需要人工介入（澄清、补充、确认）的对话比例

```
干预率 = 触发人工介入的对话数 / 总对话数 × 100%

人工介入类型:
- 意图澄清（置信度<0.85）
- 信息追问（缺失必填字段）
- 风险确认（高风险操作）
- 异常处理（工具失败无降级）
```

**健康区间**:

| 干预率 | 解读 | 状态 |
|-------|------|------|
| <10% | 边界可能过宽，Agent过于自信 | 🟠 |
| 10%-20% | 理想区间 | 🟢 |
| 20%-30% | 正常范围 | 🟡 |
| 30%-40% | 需关注优化 | 🟠 |
| >40% | 严重影响体验 | 🔴 |

**干预类型分布**:

```
目标分布:
├── 意图澄清: 40% (正常)
├── 信息追问: 35% (正常)
├── 风险确认: 20% (必要)
└── 异常处理: 5% (应降低)
```

---

### 3.4 用户接受率

**定义**: 用户采纳Agent输出结果的比例

```
接受率 = 用户接受的结果数 / 总输出结果数 × 100%

接受行为:
- 点击"确认"/"采纳"
- 直接使用（如发送邮件）
- 无修改编辑（预览后发布）

拒绝行为:
- 点击"换一版"/"重新生成"
- 大幅修改（>30%内容）
- 放弃使用
```

**目标值**: >80%

**细分场景**:

| 输出类型 | 目标接受率 | 优化方向 |
|---------|-----------|---------|
| 获客方案 | >85% | 个性化程度 |
| SEO文章 | >75% | 内容质量 |
| 开发信 | >80% | 个性化+专业性 |
| 客户画像 | >90% | 准确性 |
| 回复建议 | >85% | 上下文匹配 |

---

### 3.5 安全边界触发率

**定义**: 风险操作触发人工确认机制的比例

```
触发率 = 触发确认的风险操作数 / 总风险操作数 × 100%

风险操作包括:
- 发送邮件（不可逆）
- 删除任务（不可逆）
- 修改企业画像（高影响）
- 批量发送>100封（高风险）
```

**健康区间**:

| 触发率 | 解读 | 状态 |
|-------|------|------|
| <3% | 确认机制可能未生效 | 🔴 |
| 3%-10% | 偏低，可能过度自动 | 🟠 |
| 10%-20% | 理想区间 | 🟢 |
| >20% | 边界过严，影响效率 | 🟡 |

**关键要求**: 必须有触发记录，证明机制在生效

---

### 3.6 平均对话轮数

**定义**: 完成一个任务所需的平均对话轮数

```
平均轮数 = 总对话轮数 / 完成任务数

一轮 = 用户输入 + Agent回复
```

**目标值**: <5轮

**任务类型基准**:

| 任务类型 | 目标轮数 | 说明 |
|---------|---------|------|
| 查询进度 | 1-2轮 | 简单查询 |
| 获取线索 | 1-2轮 | 简单查询 |
| 创建任务 | 3-5轮 | 标准流程 |
| 优化内容 | 2-4轮 | 视修改次数 |
| 复杂配置 | 4-6轮 | 可接受 |

**异常监控**: >8轮的任务需分析优化

---

### 3.7 工具调用成功率

**定义**: 工具调用成功完成的比例

```
成功率 = 成功调用次数 / 总调用次数 × 100%
```

**目标值**: >95%

**工具细分**:

| 工具类型 | 目标成功率 | 降级策略 |
|---------|-----------|---------|
| 意图识别 | >99% | 规则兜底 |
| 数据查询 | >98% | 缓存降级 |
| 内容生成 | >95% | 模板降级 |
| 邮件发送 | >99% | 暂停+提示 |
| 站点发布 | >98% | 保存草稿 |

---

### 3.8 降级处理率

**定义**: 主路径失败后触发降级方案的比例

```
降级率 = 降级处理次数 / 主路径失败次数 × 100%
```

**目标值**: >90%（失败时大部分能降级）

**健康警示**: 
- 降级率>20%: 主路径稳定性需关注
- 降级率<80%: 降级覆盖不完整

---

### 3.9 置信度分布

**定义**: 意图识别置信度的分布情况

**目标分布**:

| 置信度区间 | 目标占比 | 解读 |
|-----------|---------|------|
| 0.9-1.0 | >60% | 高置信，直接执行 |
| 0.8-0.9 | ~25% | 中高置信，可执行 |
| 0.7-0.8 | ~10% | 中等置信，需确认 |
| 0.6-0.7 | ~5% | 低置信，需澄清 |
| <0.6 | <5% | 极低，无法识别 |

**监控告警**:
- <0.7占比>15% → 模型需优化
- >0.9占比<50% → 可能过于保守

---

### 3.10 用户满意度

**定义**: 用户对Agent服务的满意度评分

**测量方式**:
- 任务完成后弹窗评分（1-5星）
- NPS调研（月度）
- 用户反馈文本分析

**目标值**: NPS > 40，平均评分 > 4.2

## 5. 评估实施计划

### 5.1 数据收集

| 数据来源 | 收集方式 | 频率 |
|---------|---------|------|
| 对话日志 | 实时记录 | 实时 |
| 用户反馈 | 主动收集+被动记录 | 每次任务后 |

| 评估类型 | 频率 | 内容 |
|---------|------|------|
| 实时监控 | 实时 | 核心指标Dashboard |

| 月报 | 每月 | 深度分析+优化建议 |
| 季度评估 | 每季 | 全面评估+策略调整 |

## 6. 持续优化机制

### 6.1 问题诊断流程

```
指标异常
    │
    ▼
根因分析
    ├── 数据问题 ──► 修正数据/补充数据
    │
    ├── 模型问题 ──► 模型优化/重训练
    │
    ├── 策略问题 ──► 调整阈值/规则
    │
    └── 产品问题 ──► 需求调整/功能优化
```

### 6.2 A/B测试框架

| 测试对象 | 测试内容 | 评估指标 |
|---------|---------|---------|
| 意图模型 | 新版本vs旧版本 | 准确率、置信度分布 |
| 决策阈值 | 不同阈值配置 | 完成率、干预率 |
| 交互策略 | 不同追问方式 | 轮数、满意度 |
| 记忆权重 | 不同权重公式 | 任务完成率 |
