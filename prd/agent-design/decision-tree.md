# DealClaw Agent 决策树设计

> 版本: V1.0  
> 更新日期: 2026-03-30  
> 设计目标: 定义Agent"何时停下、何时继续、何时询问"的决策逻辑

---

## 1. 核心设计原则

### 1.1 Agent决策 vs 传统流程

| 维度 | 传统预设流程 ❌ | Agent自主决策 ✅ |
|------|---------------|----------------|
| 执行路径 | 预设流程图 | 动态决策树 |
| 分支判断 | 基于固定条件 | 基于置信度+上下文 |
| 异常处理 | 边界外的补丁 | 边界是核心设计 |
| 用户交互 | 被动响应 | 主动确认/询问 |

### 1.2 决策树核心三问

1. **何时继续？** - 信息充足、意图明确、路径清晰
2. **何时停下询问？** - 信息缺失、意图模糊、需要确认
3. **何时终止？** - 无法完成、用户取消、超出能力

## 2. 决策状态机

### 2.1 顶层状态流转

```
                    ┌─────────────────┐
                    │   等待用户输入   │
                    │  (Waiting Input)│
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
              ▼              ▼              ▼
       ┌──────────┐   ┌──────────┐   ┌──────────┐
       │  意图识别 │   │  系统指令 │   │  闲聊对话 │
       │(Intent)  │   │(Command) │   │(Chat)    │
       └────┬─────┘   └────┬─────┘   └────┬─────┘
            │              │              │
            ▼              │              │
    ┌───────────────┐      │              │
    │ 置信度<0.6?   │────Yes─────────────►请求澄清
    │               │                         │
    │ No            │◄────────────────────────┘
    └───────┬───────┘
            │
    ┌───────▼───────┐
    │ 置信度0.6-0.85?│────Yes──────────────►推测确认
    │               │                         │
    │ No            │◄────────────────────────┘
    └───────┬───────┘
            │
            ▼
    ┌───────────────┐
    │ 信息完整度检查 │────不完整──────────────►追问缺失
    │               │                         │
    │ 完整          │◄────────────────────────┘
    └───────┬───────┘
            │
            ▼
    ┌───────────────┐
    │   执行任务    │
    └───────┬───────┘
            │
    ┌───────▼───────┐
    │  工具调用     │
    │               │
    ├─► 成功        │───────────────────────►结果输出
    │               │                           │
    ├─► 失败(可降级)│───────────────────────►自动降级
    │               │                           │
    └─► 失败(严重)  │───────────────────────►暂停报错
```

### 2.2 详细状态定义

| 状态 | 描述 | 触发条件 | 退出条件 |
|-----|------|---------|---------|
| Waiting Input | 等待用户输入 | 系统就绪/上一步完成 | 收到用户输入 |
| Intent Recognition | 意图识别中 | 收到文本输入 | 识别完成 |
| Clarification | 请求澄清 | 置信度<0.6 | 用户重新输入 |
| Confirmation | 推测确认 | 置信度0.6-0.85 | 用户确认/否认 |
| Information Gathering | 信息收集 | 信息不完整 | 信息完整或用户跳过 |
| Task Execution | 任务执行 | 信息完整+意图明确 | 执行完成/失败 |
| Tool Invocation | 工具调用 | 需要外部能力 | 工具返回结果 |
| Degradation | 降级处理 | 工具失败 | 降级方案执行 |
| Result Output | 结果输出 | 任务完成 | 等待用户反馈 |
| Pause | 暂停等待 | 风险操作/用户暂停 | 用户决策 |
| Error | 错误状态 | 无法恢复的错误 | 错误处理完成 |

## 3. 决策条件详细定义

### 3.1 继续执行条件

**判定公式**:
```
继续执行 = (意图置信度 >= 0.85) 
         AND (信息完整度 >= 0.8) 
         AND (无风险操作) 
         AND (工具可用)
```

**判定逻辑**:

| 条件 | 阈值 | 判定逻辑 | 权重 |
|-----|------|---------|------|
| 意图置信度 | >=0.85 | 模型输出置信度 | 40% |
| 信息完整度 | >=0.8 | 必填槽位填充率 | 30% |
| 上下文一致性 | >=0.7 | 与历史对话连贯性 | 15% |
| 工具就绪状态 | =可用 | 所需工具状态检查 | 15% |

**执行动作**:
1. 生成执行计划
2. 调用相关工具
3. 监控执行进度
4. 准备结果输出

### 3.2 暂停询问条件

**判定公式**:
```
暂停询问 = (0.6 <= 意图置信度 < 0.85) 
         OR (信息完整度 < 0.8) 
         OR (存在歧义)
```

**询问类型与策略**:

| 暂停原因 | 判定条件 | 询问策略 | 示例 |
|---------|---------|---------|------|
| 意图模糊 | 置信度0.6-0.85 | 提供选项确认 | "您是想...对吗？" |
| 信息缺失 | 必填槽位缺失 | 聚焦追问 | "请告诉我..." |
| 存在歧义 | 多意图可能>0.3 | 列表选择 | "请选择：[1]...[2]..." |
| 时间模糊 | 时间表达式不明确 | 范围确认 | "您指的是本周还是本月？" |
| 对象模糊 | 指代词无法解析 | 上下文确认 | "您指的是刚才提到的...？" |

### 3.3 推测确认条件

**判定公式**:
```
推测确认 = (0.7 <= 意图置信度 < 0.85) 
         AND (存在最可能意图) 
         AND (推测置信度 > 0.6)
```

**推测确认流程**:

```
[意图识别结果]
├── 意图A: 0.45 (创建任务)
├── 意图B: 0.35 (查询进度) ← 最可能
└── 意图C: 0.20 (其他)

Agent: "根据您的描述，我推测您是想**查询任务进度**。
       
       如果不是，请告诉我您想：
       [1] 创建新的获客任务
       [2] 其他（请描述）
       
       请回复'是'或选择数字。"
```

### 3.4 终止任务条件

**终止类型与判定**:

| 终止类型 | 判定条件 | 终止动作 | 用户反馈 |
|---------|---------|---------|---------|
| 无法完成 | 超出产品能力范围 | 保存草稿（如有） | "我目前还不能..." |
| 用户取消 | 明确表达取消 | 清理临时状态 | "已取消，随时可以继续" |
| 资源不足 | 数据/计算资源不足 | 记录需求 | "当前无法处理，已记录" |
| 严重错误 | 不可恢复的系统错误 | 错误日志 | "遇到技术问题，请稍后重试" |
| 超时放弃 | 用户长时间无响应 | 保存状态 | "会话超时，已保存进度" |

### 3.5 降级处理条件

**判定公式**:
```
降级处理 = (主路径工具失败) 
         AND (存在降级方案) 
         AND (降级方案可用)
```

**降级策略矩阵**:

| 主路径失败 | 降级方案 | 用户告知 | 质量影响 |
|-----------|---------|---------|---------|
| AI生成超时 | 使用模板+人工填充 | "使用标准模板生成" | 个性化度降低 |
| 企业库查询超时 | 使用缓存数据 | "使用缓存数据，可能不是最新" | 实时性降低 |
| 关键词库无匹配 | 使用通用关键词 | "使用推荐关键词" | 精准度降低 |
| 质量检测失败 | 标记为待审核 | "已保存，建议人工审核" | 风险增加 |
| 站点发布失败 | 保存草稿 | "保存至草稿箱，请检查配置" | 延迟发布 |

## 5. 边界条件决策

### 5.1 用户干预处理

| 干预时机 | 当前状态 | 处理策略 | 状态转移 |
|---------|---------|---------|---------|
| 任务执行中改需求 | Task Execution | 保存进度→询问重启或继续 | 暂停→重新规划 |
| 用户纠正理解 | any | 立即修正→更新记忆→道歉 | 继续执行 |
| 用户要求暂停 | any | 立即停止→保存状态→报告 | Pause状态 |
| 用户要求回退 | any | 回退到上一确认节点 | 回溯状态 |
| 用户切换话题 | any | 保存当前上下文→切换 | 新意图识别 |

### 5.2 工具失败决策

```
[工具调用]
    │
    ▼
[调用结果]
├── 成功 ──────────────────────► 继续执行
│
├── 超时
│   ├── 有缓存数据 ──► 使用缓存 ──► 提示用户
│   └── 无缓存数据 ──► 重试1次 ──► 仍失败 ──► 降级/暂停
│
├── API错误(5xx)
│   ├── 有备用API ──► 切换备用 ──► 静默降级
│   └── 无备用API ──► 重试 ──► 降级/暂停
│
├── 数据缺失
│   ├── 可扩大条件 ──► 放宽筛选 ──► 提示用户
│   └── 不可扩大 ──► 暂停询问
│
└── 权限错误
    ├── 可降级 ──► 降级执行
    └── 不可降级 ──► 暂停询问
```

### 5.3 置信度波动处理

```
[连续对话中]
    │
    ▼
[第N轮意图识别]
├── 置信度稳定(±0.1) ──► 正常执行
│
├── 置信度骤降(>0.2)
│   ├── 上下文突变? ──► 重新理解
│   └── 用户纠正? ──► 更新理解
│
└── 置信度波动频繁
    ├── 检查实体一致性
    ├── 提示"我可能对...理解有误"
    └── 请求明确确认
```

## 6. 决策阈值配置

### 6.1 全局阈值

| 阈值名称 | 默认值 | 可调范围 | 说明 |
|---------|-------|---------|------|
| 直接执行置信度 | 0.85 | 0.8-0.95 | 高于此值直接执行 |
| 推测确认置信度 | 0.70 | 0.6-0.8 | 此区间需要推测确认 |
| 请求澄清置信度 | 0.60 | 0.5-0.7 | 低于此值请求澄清 |
| 信息完整度阈值 | 0.80 | 0.7-0.9 | 高于此值认为信息完整 |
| 上下文一致性阈值 | 0.70 | 0.6-0.8 | 话题一致性检查 |
| 风险操作置信度 | 0.90 | 0.85-0.95 | 风险操作需更高置信度 |

### 6.2 按意图类型的阈值调整

| 意图类型 | 直接执行阈值 | 特殊规则 |
|---------|-------------|---------|
| 创建任务 | 0.85 | 信息完整度>0.7可降级执行 |
| 查询进度 | 0.75 | 低置信度时可展示全部 |
| 删除任务 | 0.95 | 必须双重确认，无视置信度 |
| 修改画像 | 0.90 | 必须确认影响范围 |
| 获取线索 | 0.75 | 低置信度时展示最近 |

## 7. 监控与评估

### 7.1 决策质量指标

| 指标 | 目标值 | 监控方式 |
|-----|-------|---------|
| 意图识别准确率 | >90% | 抽样标注 |
| 推测确认接受率 | >75% | 用户反馈 |
| 追问转化率 | >80% | 对话统计 |
| 误执行率 | <5% | 用户纠正 |

### 7.2 决策异常告警

| 异常类型 | 触发条件 | 告警级别 |
|---------|---------|---------|
| 置信度异常分布 | <0.7占比>20% | 警告 |
| 频繁澄清 | 单会话澄清>3次 | 提示 |
| 推测被拒 | 接受率<50% | 警告 |
| 执行回退 | 用户回退>10% | 严重 |

## 8. 自检清单

### 决策树设计完成标准

- [x] 定义了完整的决策状态机
- [x] 每个决策点有明确的判定条件和阈值
- [x] 多轮对话流程覆盖主要场景
- [x] 风险操作有强制确认机制
- [x] 工具失败有降级处理策略
- [x] 阈值可配置，支持按意图调整

### 边界条件验证

1. **Agent知道何时该停下来吗？**
   - ✅ 置信度<0.6时停下来请求澄清
   - ✅ 信息缺失时停下来追问
   - ✅ 风险操作前停下来等待确认
   - ✅ 工具失败时停下来降级或报错

2. **决策路径清晰可追溯吗？**
   - ✅ 每个决策记录理由和置信度
   - ✅ 支持回退到上一决策点
   - ✅ 用户可查看决策依据

3. **降级方案都有覆盖吗？**
   - ✅ 所有核心工具有降级方案
   - ✅ 降级时告知用户质量影响
   - ✅ 严重失败时有暂停机制
