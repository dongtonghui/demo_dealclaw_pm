# DealClaw 意图建模体系设计

> 版本: V1.0  
> 更新日期: 2026-03-30  
> 设计原则: 从"功能描述"到"意图理解"的范式转换

---

## 1. 核心思维转变

### 1.1 传统PRD vs Agent-native设计

| 维度 | 传统PRD思维 ❌ | Agent-native思维 ✅ |
|------|---------------|-------------------|
| 起点 | 这个功能是什么？ | 用户带着什么**意图**来？ |
| 设计重点 | 预设流程图 | 动态**意图识别**+多轮确认 |
| 成功标准 | 功能完整可用 | 准确理解用户**真实需求** |
| 异常处理 | 边界外的if-else | **歧义消解**是核心设计 |

### 1.2 意图建模三问

在定义每个意图前，必须回答：

1. **触发场景**：用户在什么处境下表达这个意图？（时间压力？信息缺口？）
2. **成功标准**：用户心里的"任务完成"长什么样？
3. **容忍边界**：用户能接受Agent犯什么错？哪些绝对不能犯？

## 3. 核心意图详细定义

### 3.1 创建获客任务（Create Task）

#### 基本信息

| 属性 | 值 |
|-----|---|
| 意图ID | INTENT_CREATE_TASK |
| 优先级 | P0 |
| 置信度阈值 | 0.75（触发多轮确认）/ 0.85（直接执行） |
| 风险等级 | 低 |

#### 触发场景

| 场景 | 示例表达 | 触发条件 |
|-----|---------|---------|
| 明确获客需求 | "帮我找美国户外用品批发商" | 关键词：找/寻找/开发/获客 + 地区 + 行业 |
| 模糊需求表达 | "我想拓展美国市场" | 关键词：拓展/进入/开发 + 市场/客户 |
| 竞品参考表达 | "帮我找像REI这样的客户" | 关键词：像/类似 + 公司名 |
| 渠道指定 | "发邮件给美国批发商" | 关键词：发邮件/联系 + 地区 + 客户类型 |

#### 实体槽位定义

| 槽位 | 类型 | 必填 | 提取规则 | 示例 |
|-----|------|-----|---------|------|
| target_region | 地区 | 是 | NER+关键词匹配 | 美国/USA/北美 |
| industry | 行业 | 是 | 关键词库+语义匹配 | 户外用品/电子产品 |
| customer_type | 客户类型 | 否 | 关键词匹配 | 批发商/品牌商/零售商 |
| company_size | 规模 | 否 | 范围提取 | 50-200人/中大型 |
| annual_purchase | 采购额 | 否 | 数值+单位提取 | 100万-500万美元 |
| timeline | 时间预期 | 否 | 时间表达式解析 | 3个月内/本季度 |
| priority_keywords | 关键词 | 否 | 名词提取 | 帐篷/露营装备 |

#### 成功标准

用户认可的判断标准：
1. 生成的客户画像卡片准确反映需求
2. 获客方案符合预期
3. 任务成功启动并收到执行确认

#### 容忍边界

| 边界条件 | 处理策略 |
|---------|---------|
| 地区表达模糊（如"国外"） | 暂停询问，提供选项 |
| 行业不明确 | 基于企业画像推测+确认 |
| 缺少时间预期 | 使用默认值（3个月）+ 提示 |
| 客户类型缺失 | 使用"批发商"默认值 + 可编辑 |

#### 多轮确认策略

```
场景1: 信息基本完整（完整度>0.8）
用户: "帮我找美国户外用品批发商"
Agent: "收到！我来为您创建获客任务。
       基于您的企业画像（户外用品出口商）：
       • 目标地区：美国
       • 行业：户外用品
       • 客户类型：批发商
       
       让我确认几个细节：
       1. 目标客户年采购额大概？
       2. 期望多长时间看到效果？"

场景2: 信息缺失较多（完整度0.5-0.8）
Agent: "好的！我来帮您创建获客任务。
       您提到：美国市场 + 户外用品
       
       需要确认：
       1. 您想找哪类客户？[批发商/品牌商/零售商]
       2. 目标客户的规模？
       3. 期望的获客时间？"

场景3: 意图模糊（完整度<0.5）
Agent: "您是想创建一个新的获客任务吗？
       
       我可以帮您：
       [1] 寻找特定地区的潜在客户
       [2] 启动邮件外联 campaign
       [3] 制定SEO内容策略
       
       请告诉我您的具体需求。"
```

## 4. 模糊意图处理策略

### 4.1 模糊意图识别

| 模糊类型 | 识别特征 | 置信度范围 |
|---------|---------|-----------|
| 代词指代不明 | 包含"它"/"这个"/"那个" | 0.5-0.7 |
| 动作对象缺失 | 只有动词，没有宾语 | 0.4-0.6 |
| 程度副词模糊 | "一些"/"一点"/"优化一下" | 0.5-0.7 |
| 多意图混淆 | 包含多个动词/目标 | 0.6-0.75 |
| 领域外意图 | 超出产品能力范围 | <0.5 |

### 4.2 多轮确认流程

```
[用户输入: "帮我优化一下"]
         │
         ▼
[意图识别] → 置信度: 0.45（多意图可能）
         │
         ▼
[上下文分析]
  ├── 当前对话: 刚完成获客任务创建
  ├── 当前任务: 美国户外用品批发商（进行中）
  ├── 最近操作: 查看了邮件发送效果
  └── 历史偏好: 经常优化邮件内容
         │
         ▼
[生成候选意图列表]
  1. 优化获客任务配置 (0.35)
  2. 优化邮件内容 (0.30)
  3. 优化SEO文章 (0.25)
  4. 其他 (0.10)
         │
         ▼
[展示推测+请求确认]
Agent: "您是想优化以下内容吗？
       
       [1] 当前获客任务的配置
       [2] 最近发送的开发信内容
       [3] 已发布的SEO文章
       [4] 其他（请描述）
       
       请回复数字选择。"
         │
         ▼
[用户选择] → 确认意图 → 执行
```

### 4.3 上下文回溯策略

当用户输入包含指代词时，回溯范围：

| 指代词 | 回溯范围 | 示例 |
|-------|---------|------|
| "它"/"这个" | 当前对话最近3轮 | 指代刚提到的任务/线索 |
| "那个" | 当前会话最近提及的同类对象 | 指代某个特定任务 |
| "这些"/"那些" | 当前会话相关列表 | 指代一组线索 |
| "刚才说的" | 当前对话前5轮 | 指代之前的某个话题 |

## 6. 置信度与决策映射

### 6.1 意图识别置信度分级

| 置信度区间 | 决策 | 用户反馈 |
|-----------|------|---------|
| 0.9 - 1.0 | 直接执行 | 正常响应 |
| 0.85 - 0.9 | 执行+确认 | "已为您..." |
| 0.75 - 0.85 | 推测确认 | "您是想...对吗？" |
| 0.6 - 0.75 | 多轮询问 | "请问您是想...？" |
| 0.4 - 0.6 | 请求澄清 | "抱歉，我没理解，您能否..." |
| < 0.4 | 超出能力 | "我目前还不能处理..." |

### 6.2 信息完整度评估

```python
def calculate_completeness(intent, filled_slots, required_slots):
    """
    计算信息完整度
    """
    if not required_slots:
        return 1.0
    
    filled_count = sum(1 for slot in required_slots if slot in filled_slots)
    base_completeness = filled_count / len(required_slots)
    
    # 有默认值的可选槽位给予部分分数
    optional_with_default = [s for s in filled_slots if s not in required_slots]
    bonus = len(optional_with_default) * 0.05
    
    return min(1.0, base_completeness + bonus)
```

### 6.3 综合决策矩阵

| 意图置信度 | 信息完整度 | 决策 | 策略 |
|-----------|-----------|------|------|
| >0.85 | >0.8 | 直接执行 | 正常流程 |
| >0.85 | 0.5-0.8 | 追问缺失 | 聚焦缺失信息 |
| >0.85 | <0.5 | 推测+追问 | 基于上下文推测 |
| 0.7-0.85 | >0.8 | 推测确认 | "您是想...对吗？" |
| 0.7-0.85 | <0.8 | 多轮确认 | 意图+信息双重确认 |
