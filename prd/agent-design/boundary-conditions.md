# DealClaw 边界条件设计

> 版本: V1.0  
> 更新日期: 2026-03-30  
> 核心理念: **边界条件设计是核心，不是补丁**

---

## 1. 边界条件设计哲学

### 1.1 核心转变

| 传统思维 ❌ | Agent-native思维 ✅ |
|------------|-------------------|
| 异常处理是功能完成后的补丁 | 边界条件是核心设计 |
| 假设用户输入清晰明确 | 假设意图天然模糊 |
| 预设所有执行路径 | 定义决策条件让Agent自主规划 |
| 错误=系统故障 | 边界触发=正常行为 |

### 1.2 边界条件三问

写PRD前必答：

1. **这个Agent在什么情况下应该停下来？**
2. **如果它做了一个错误的决策，用户能发现吗？**
3. **有没有哪个动作，是它绝对不能在没有人工确认的情况下执行的？**

## 2. 边界条件分类体系

### 2.1 五大边界类型

```
边界条件
├── 意图歧义边界（Ambiguity Boundary）
│   └── 用户表达不明确、存在多种理解可能
│
├── 工具失败边界（Tool Failure Boundary）
│   └── 外部依赖失效、数据不可用、服务超时
│
├── 结果不确定边界（Uncertainty Boundary）
│   └── Agent对自身输出置信度不足
│
├── 用户干预边界（Intervention Boundary）
│   └── 用户主动改变需求、纠正错误、要求暂停
│
└── 风险操作边界（Risk Boundary）
    └── 不可逆操作、高影响操作、需要授权的操作
```

## 3. 意图歧义边界

### 3.1 歧义类型矩阵

| 歧义类型 | 典型示例 | 识别特征 | 处理策略 |
|---------|---------|---------|---------|
| **代词指代不明** | "帮我优化它" | 包含"它"/"这个"/"那个" | 上下文回溯+确认 |
| **动作对象模糊** | "发送出去" | 只有动词，无明确宾语 | 确认目标列表 |
| **程度副词模糊** | "增加一些" | "一些"/"一点"/"优化一下" | 量化确认 |
| **时间表达模糊** | "最近的效果" | 相对时间无明确范围 | 时间范围选项 |
| **多任务混淆** | "查看并优化" | 多个动词/目标 | 分解为子任务确认顺序 |
| **领域模糊** | "设置一下" | 动作明确但领域不明 | 确认操作对象 |

### 3.2 代词指代消解

**回溯范围规则**:

| 指代词 | 回溯范围 | 匹配优先级 | 示例 |
|-------|---------|-----------|------|
| "它"/"这个" | 当前对话最近3轮 | 最近提及的同类对象 | 指代刚提到的任务 |
| "那个" | 当前会话最近提及 | 用户特指的某个对象 | 指代某个特定任务 |
| "这些"/"那些" | 当前会话相关列表 | 最近展示的列表 | 指代一组线索 |
| "刚才说的" | 当前对话前5轮 | 语义相关性 | 指代之前的话题 |
| "之前" | 历史会话（24h内） | 时间+相关性 | 指代之前的任务 |

**消解失败处理**:
```
用户: "帮我优化它"
Agent: "您指的是以下哪一项？
       
       [1] 正在进行的获客任务（美国户外用品）
       [2] 昨天发送的开发信
       [3] 上周发布的SEO文章
       
       请回复数字，或告诉我具体名称。"
```

### 3.3 程度副词量化

| 用户表达 | 量化映射 | 确认策略 |
|---------|---------|---------|
| "一些"/"一点" | 当前值的10-20% | "增加约15%可以吗？" |
| "很多"/"大幅" | 当前值的50%+ | "增加到原来的1.5倍？" |
| "稍微"/"略微" | 当前值的5-10% | "微调5%左右？" |
| "尽量"/"最大" | 系统上限或用户上限 | "使用最高配置？" |
| "默认"/"标准" | 系统预设值 | 直接应用+告知 |

### 3.4 时间表达标准化

| 用户表达 | 标准化范围 | 确认方式 |
|---------|-----------|---------|
| "最近" | 最近7天 | 不确认，直接使用 |
| "这周" | 本周一至今天 | 不确认，直接使用 |
| "本月" | 本月1日至今天 | 不确认，直接使用 |
| "前段时间" | 最近30天 | "您指的是最近一个月吗？" |
| "之前" | 需要上下文 | "您具体指的是哪个时间段？" |
| "去年" | 去年全年 | 不确认，直接使用 |

## 4. 工具失败边界

### 4.1 失败分类与处理

| 失败类型 | 触发条件 | 自动降级方案 | 用户通知 | 日志级别 |
|---------|---------|-------------|---------|---------|
| **超时** | 超过设定阈值 | 使用缓存/简化版 | 轻提示 | Warning |
| **API错误** | 5xx错误 | 切换备用API | 无 | Error |
| **限流** | 429错误 | 指数退避重试 | 延迟提示 | Warning |
| **数据缺失** | 返回空结果 | 放宽筛选条件 | 轻提示 | Info |
| **格式异常** | 返回非预期格式 | 使用默认格式 | 无 | Error |
| **权限不足** | 403错误 | 暂停+提示 | 明确提示 | Error |
| **依赖故障** | 下游服务故障 | 暂停+提示 | 明确提示 | Critical |

### 4.2 超时处理策略

```
[工具调用]
    │
    ▼
[超时检测]
    │
    ├──► 有缓存数据 ──► 使用缓存 ──► 提示"可能不是最新"
    │
    ├──► 有简化方案 ──► 执行简化版 ──► 提示"使用简化版"
    │
    ├──► 可异步获取 ──► 先返回占位 ──► 后台获取+推送更新
    │
    └──► 无降级方案 ──► 暂停 ──► 询问用户"重试/跳过/取消"
```

**超时时间配置**:

| 工具类型 | 超时时间 | 降级策略 |
|---------|---------|---------|
| 意图识别 | 2s | 规则匹配 |
| 实体提取 | 1s | 正则匹配 |
| 数据查询 | 5s | 缓存数据 |
| 内容生成 | 30s | 提供框架 |
| 邮件发送 | 5s | 暂停+提示 |
| 站点发布 | 10s | 保存草稿 |

### 4.3 熔断机制

**熔断器状态流转**:

```
[Closed] ──► 失败计数++ ──► 失败率>阈值 ──► [Open]
    ▲                                        │
    │                                        │ 超时后
    │                                        ▼
    └──── 成功率>阈值 ───── [Half-Open] ◄───┘
```

**熔断配置**:

| 工具 | 失败率阈值 | 熔断时长 | 半开试探次数 |
|-----|-----------|---------|------------|
| 企业库查询 | 50% | 30s | 3次 |
| 邮件发送 | 30% | 60s | 5次 |
| AI生成 | 20% | 120s | 2次 |

## 5. 结果不确定边界

### 5.1 置信度分级表达

| 置信度区间 | 表达策略 | 示例 |
|-----------|---------|------|
| **0.9 - 1.0** | 直接回答，不标注 | "已为您生成获客方案..." |
| **0.8 - 0.9** | 正常回答+开放结尾 | "...如有疑问请告诉我" |
| **0.7 - 0.8** | 回答+置信标注 | "根据现有信息，我认为..." |
| **0.6 - 0.7** | 推测性表达 | "这个我不太确定，但推测可能是..." |
| **< 0.6** | 不回答，请求澄清 | "抱歉，我没理解您的意思..." |

### 5.2 不确定性表达方式

**方式1: 置信度显式标注**
```
Agent: "基于数据分析，我有85%的把握认为：
       本周获客效果比上周提升了23%。"
```

**方式2: 可能性范围表达**
```
Agent: "根据目前数据，您的获客成本可能在：
       ¥80-120/条之间（置信区间90%）"
```

**方式3: 多结果并列**
```
Agent: "有两种可能的理解：
       A. 您想查询美国任务的进展（概率70%）
       B. 您想查询全部任务的概览（概率25%）
       
       请确认或选择。"
```

**方式4: 推理过程透明**
```
Agent: "我得出这个结论的依据是：
       1. 邮件打开率28%（行业平均25%）✓
       2. 回复率4.2%（高于您的历史平均3.5%）✓
       3. 但退信率2%略高于理想值1% ⚠️
       
       综合来看，效果良好但有优化空间。"
```

### 5.3 不确定场景的交互策略

| 场景 | 不确定点 | Agent响应策略 |
|-----|---------|--------------|
| 线索评分边界 | 评分79 vs 80（高意向阈值） | "此线索评分79分，接近高意向标准，建议关注" |
| 任务完成预测 | 完成时间有波动 | "预计3-4周完成，取决于市场反馈" |
| 效果归因 | 无法确定具体原因 | "可能的原因包括：A... B... 需要更多数据确认" |
| 客户意向判断 | 信号矛盾 | "该客户表现出混合信号，建议进一步沟通确认" |

## 6. 用户干预边界

### 6.1 干预时机与处理

| 干预时机 | 当前状态 | 处理策略 | 状态转移 |
|---------|---------|---------|---------|
| **任务执行中改需求** | Task Execution | 保存进度→询问重启或继续 | 暂停→重新规划 |
| **用户纠正理解错误** | any | 立即修正→更新记忆→道歉 | 继续执行 |
| **用户要求暂停** | any | 立即停止→保存状态→报告进度 | Pause状态 |
| **用户要求回退** | any | 回退到上一确认节点 | 状态回溯 |
| **用户切换话题** | any | 保存当前上下文→切换 | 新意图识别 |
| **用户要求加速** | Task Execution | 评估影响→调整参数→确认 | 调整执行 |
| **用户要求跳过** | Information Gathering | 记录跳过→使用默认值→继续 | 继续执行 |

### 6.2 需求变更处理

**场景**: 用户创建任务后，立即修改目标客户画像

```
用户: "帮我找美国户外用品批发商"
    │
    ▼
[创建任务流程中]
    │
用户: "等等，改成欧洲的"
    │
    ▼
[Agent检测需求变更]
    │
    ▼
Agent: "收到！您想将目标地区从美国改为欧洲。
       
       当前已收集的信息：
       • 地区：美国 → **欧洲**
       • 行业：户外用品（不变）
       • 客户类型：批发商（不变）
       
       [1] 基于欧洲重新生成方案
       [2] 只修改地区，其他保持不变
       [3] 重新开始创建"
    │
    ▼
[用户选择] → 执行对应逻辑
```

### 6.3 取消与回退

**取消层级**:

| 取消层级 | 触发方式 | 影响范围 | 恢复方式 |
|---------|---------|---------|---------|
| 当前步骤 | "取消"/"算了" | 当前操作 | 重新触发 |
| 当前任务 | "放弃这个任务" | 本次任务 | 从草稿恢复 |
| 全部重置 | "重新开始" | 本次会话 | 从Onboarding恢复 |

**回退机制**:
```
[当前状态]
    │
    ▼
[用户: "回到上一步"]
    │
    ▼
[查找上一确认节点]
├── 画像确认节点 ──► 恢复到画像编辑前
├── 方案确认节点 ──► 恢复到方案生成前
└── 执行确认节点 ──► 恢复到执行前
```

## 7. 风险操作边界

### 7.1 风险操作分类

| 风险等级 | 操作类型 | 影响 | 确认级别 |
|---------|---------|------|---------|
| **🔴 极高** | 批量发送>100封邮件 | 域名信誉风险 | 双重确认+验证码 |
| **🔴 极高** | 删除进行中任务 | 数据永久丢失 | 双重确认+文字输入 |
| **🟠 高** | 发送邮件（单条） | 不可逆、客户印象 | 预览+确认 |
| **🟠 高** | 修改企业画像 | 影响后续所有任务 | 影响预览+确认 |
| **🟡 中** | 发布文章到站点 | 公开可见、SEO影响 | 预览+确认 |
| **🟡 中** | 删除已完成任务 | 历史记录丢失 | 单条确认 |
| **🟢 低** | 暂停/恢复任务 | 可逆 | 单条确认 |
| **🟢 低** | 导出数据 | 无影响 | 无需确认 |

### 7.2 确认机制设计

**单条确认（单层）**:
```
Agent: "确认发布文章到您的独立站？
       
       [预览卡片]
       
       [确认发布] [取消]"
```

**双重确认（两层）**:
```
Agent: "⚠️ 确认删除任务
       
       任务：美国户外用品批发商
       影响：23条线索将被移出任务（保留在线索箱）
       
       [确认删除] [取消]"
       │
用户点击"确认删除"
       │
       ▼
Agent: "安全确认
       
       请输入"删除"二字以确认：
       [输入框]
       
       [确认] [取消]"
```

**验证码确认（高风险）**:
```
Agent: "⚠️ 高风险操作确认
       
       您即将发送邮件给156个收件人
       这可能会影响您的域名信誉
       
       请输入验证码 **4582** 确认：
       [输入框]
       
       [确认发送] [取消]"
```

### 7.3 防误触设计

| 场景 | 防误触策略 |
|-----|-----------|
| 按钮位置 | 确认按钮与取消按钮间距充足 |
| 二次确认 | 高风险操作必须二次确认 |
| 冷却时间 | 连续操作间隔>2秒 |
| 撤销窗口 | 发送后30秒内可撤回 |
| 操作记录 | 所有风险操作记录审计日志 |

## 8. 边界触发监控

### 8.1 边界触发率指标

| 边界类型 | 健康阈值 | 告警阈值 | 监控方式 |
|---------|---------|---------|---------|
| 意图歧义触发 | 15-30% | >40% | 意图识别日志 |
| 工具降级触发 | <10% | >20% | 工具调用日志 |
| 不确定表达触发 | 10-20% | >30% | 输出分析 |
| 用户干预触发 | <15% | >25% | 会话分析 |
| 风险确认触发 | >5% | <3% | 操作日志 |

> 注：风险确认触发率<3%可能意味着确认机制未生效

### 8.2 边界处理质量评估

| 指标 | 目标值 | 评估方式 |
|-----|-------|---------|
| 歧义消解成功率 | >80% | 用户确认率 |
| 降级方案接受率 | >90% | 用户反馈 |
| 误触发率 | <5% | 用户投诉 |
| 用户满意度 | >4.0/5 | 满意度调查 |

### 边界条件设计完成标准

- [x] 风险操作有明确的确认机制
- [x] 所有边界条件有监控指标

### 三问验证

**Q1: 这个Agent在什么情况下应该停下来？**
- ✅ 意图置信度<0.6时，停下来请求澄清
- ✅ 信息缺失必填字段时，停下来追问
- ✅ 工具调用失败且无降级方案时，停下来报错
- ✅ 检测到风险操作时，停下来等待确认
- ✅ 用户明确要求暂停时，立即停止

**Q2: 如果它做了一个错误的决策，用户能发现吗？**
- ✅ 所有输出附带置信度标识（<0.8标注）
- ✅ 关键决策提供推理过程
- ✅ 结果可通过卡片查看和编辑
- ✅ 风险操作前有预览确认
- ✅ 支持回退和撤销

**Q3: 有没有哪个动作，是绝对不能在没有人工确认的情况下执行的？**
- ✅ 发送邮件（不可逆）
- ✅ 删除进行中任务（不可逆+数据丢失）
- ✅ 修改企业画像（高影响）
- ✅ 批量发送>100封邮件（高风险）
