# DealClaw 获客师 - 对话式数字员工平台

> 文档版本: V2.0（对话式架构重构版）  
> 最后更新: 2026-03-26  
> 核心理念: 以自然语言对话为核心，多Agent协同获客

---

## 1. 产品定位

### 1.1 一句话描述

**DealClaw 获客师**是一个对话式AI数字员工平台，用户通过自然语言与获客师对话，获客师自动拆解目标、协调SEO/邮件/社媒等Agent团队执行获客任务，并以对话形式汇报进展和交付结果。

### 1.2 核心交互模式

| 维度 | 传统SaaS | DealClaw对话式 |
|-----|---------|---------------|
| **主入口** | 功能菜单导航 | 自然语言对话 |
| **创建任务** | 填写表单 | 直接说需求 |
| **查看进度** | 进入仪表盘 | 对话中询问 |
| **获取结果** | 导出报表 | 对话中推送 |
| **操作方式** | 点击/输入 | 说/问/确认 |

### 1.3 产品形态

类似 **Kimi Claw** 的三栏式对话界面：
- **左侧**：任务/项目导航
- **中间**：对话区域（核心交互区）
- **右侧**：上下文面板（数据/线索/状态）

## 2. 核心概念

### 2.1 Agent团队架构

```
                    用户
                     │
                     ▼
            ┌─────────────────┐
            │   获客师（主Agent）│  ← 统一入口，理解需求，协调团队
            │  - 意图理解       │
            │  - 目标拆解       │
            │  - 团队协调       │
            │  - 结果汇报       │
            └────────┬────────┘
                     │
        ┌────────────┼────────────┐
        ▼            ▼            ▼
┌──────────┐  ┌──────────┐  ┌──────────┐
│ SEO Agent│  │Email     │  │社媒Agent  │
│          │  │  Agent   │  │          │
│•关键词分析│  │•邮件生成  │  │•LinkedIn │
│•内容生成  │  │•发送策略  │  │•Facebook │
│•建站优化  │  │•效果追踪  │  │•内容发布  │
└──────────┘  └──────────┘  └──────────┘
```

### 2.2 对话即界面（CUI - Conversational UI）

用户通过自然语言完成所有核心操作：

| 用户意图 | 对话示例 | 系统响应 |
|---------|---------|---------|
| 创建获客任务 | "帮我找美国户外用品批发商" | 获客师追问细节 → 生成画像 → 调用Agent制定方案 |
| 查看进度 | "我的获客任务进展如何？" | 获客师汇报各Agent工作状态和关键数据 |
| 获取线索 | "今天有什么新线索？" | 获客师推送高意向线索卡片 |
| 优化内容 | "帮我优化这封邮件" | Email Agent生成优化建议卡片 |
| 查看数据 | "本周获客效果如何？" | 获客师展示数据看板卡片 |

### 2.3 富媒体卡片

对话中嵌入可交互的富媒体卡片，替代传统表单和仪表盘：

- **画像确认卡片**：展示解析后的画像，可编辑确认
- **渠道方案卡片**：展示推荐渠道组合，可一键确认或调整
- **内容预览卡片**：展示生成的内容，可编辑、预览、发布
- **线索摘要卡片**：展示关键线索信息，可查看详情、一键跟进
- **数据看板卡片**：展示核心指标，可下钻查看详情

### 3.2 功能清单

#### 对话核心（P0）

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 自然语言对话 | 用户通过文字与获客师交流 | P0 |
| 意图识别 | 识别用户想做什么（创建任务、查进度、看线索等） | P0 |
| 实体提取 | 从对话中提取画像参数（地区、行业、规模等） | P0 |
| 上下文记忆 | 记住多轮对话的上下文 | P0 |
| 富媒体卡片 | 对话中展示可交互的卡片组件 | P0 |
| Agent切换 | @指定Agent进行专业对话 | P0 |

| Agent | 核心能力 | 优先级 |
|-------|---------|-------|
| 获客师 | 需求理解、目标拆解、团队协调、结果汇报 | P0 |

| Email Agent | 邮件生成、发送策略、效果追踪、A/B测试 | P0 |
| 社媒Agent | 内容创作、发布调度、互动管理、社媒预览 | P0 |

#### 任务管理（P0）

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 任务创建 | 对话中创建获客任务 | P0 |
| 画像生成 | 自动/半自动生成目标客户画像 | P0 |
| 渠道方案 | Agent自动生成渠道组合方案 | P0 |
| 任务执行 | 各Agent协同执行获客任务 | P0 |
| 进度汇报 | 定时/按需汇报任务进展 | P0 |

#### 线索中心（P0）

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 线索捕获 | 自动捕获各渠道线索 | P0 |
| 线索评分 | AI评估线索质量和意向 | P0 |
| 线索推送 | 高意向线索主动推送到对话 | P0 |
| 线索跟进 | 对话中一键查看详情、生成回复 | P0 |

#### 数据洞察（P1）

| 功能 | 说明 | 优先级 |
|-----|------|-------|
| 实时面板 | 右侧面板展示实时数据 | P1 |

## 6. 关键实现细节

### 6.1 对话状态管理

```
对话状态流转：

空闲 ──► 理解需求 ──► 追问细节 ──► 生成画像 ──► 确认画像 ──► 制定方案
  ▲                                              │                │
  │                                              ▼                ▼
  └──────── 汇报进度 ◄──── 执行中 ◄──── 确认方案 ◄──── 调整方案 ◄──┘
```

### 6.2 Agent协调机制

```
获客师（协调者）
    │
    ├──► 分析用户需求
    │
    ├──► 确定需要哪些Agent参与
    │
    ├──► 并行/串行调用Agent
    │     ├── SEO Agent.analyzeKeywords()
    │     ├── Email Agent.getTargetList()
    │     └── 社媒Agent.prepareContent()
    │
    ├──► 汇总Agent输出
    │
    └──► 生成统一方案汇报给用户
```

### 6.3 上下文记忆

```
对话上下文包括：
- 当前任务ID
- 已确认的画像参数
- 已选择的渠道配置
- 最近的Agent输出
- 用户的偏好设置

上下文用于：
- 多轮对话连贯性
- 追问时参考历史
- Agent协作时共享信息
```

## 8. 技术架构建议

### 8.1 前端架构

```
React + TypeScript + Tailwind CSS
│
├── 布局组件
│   ├── Sidebar（左侧导航）
│   ├── ChatArea（对话区域）
│   └── ContextPanel（右侧面板）
│
├── 对话组件
│   ├── MessageList（消息列表）
│   ├── MessageBubble（消息气泡）
│   ├── RichCard（富媒体卡片）
│   └── InputBox（输入框）
│
├── 卡片组件
│   ├── ProfileCard（画像卡片）
│   ├── StrategyCard（方案卡片）
│   ├── ContentCard（内容卡片）
│   ├── LeadCard（线索卡片）
│   └── DashboardCard（数据卡片）
│
└── 状态管理
    └── Zustand / Redux Toolkit
```

### 8.2 后端架构

```
Node.js / Python FastAPI
│
├── API Gateway
│   └── 统一入口，鉴权，限流
│
├── 对话服务
│   ├── IntentClassifier（意图识别）
│   ├── EntityExtractor（实体提取）
│   ├── ContextManager（上下文管理）
│   └── ResponseGenerator（回复生成）
│
├── Agent调度中心
│   ├── AgentRegistry（Agent注册表）
│   ├── TaskOrchestrator（任务编排）
│   └── ResultAggregator（结果聚合）
│
├── Agent服务
│   ├── SEO Agent Service
│   ├── Email Agent Service
│   └── Social Agent Service
│
└── 数据层
    ├── PostgreSQL（主数据）
    ├── Redis（缓存/会话）
    └── MongoDB（对话记录）
```

### 8.3 AI能力

```
大语言模型（LLM）
│
├── 意图识别
│   └── 分类：create_task / check_progress / view_leads / optimize_content / ...
│
├── 实体提取
│   └── 画像参数：region, industry, company_size, budget, ...
│
├── 回复生成
│   └── 根据上下文生成自然、专业的回复
│
├── Agent协调
│   └── 决定调用哪些Agent，如何组合结果
│
└── 上下文记忆
    └── 维护多轮对话的连贯性
```
