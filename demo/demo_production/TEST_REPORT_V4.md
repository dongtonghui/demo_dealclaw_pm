#### Bug-001: 空消息可被发送 [HIGH] 🟠

| 属性 | 内容 |
|-----|------|
| **ID** | Bug-001 |
| **标题** | 空消息和仅空格消息可被发送 |
| **级别** | 🟠 High |
| **模块** | ChatPanel |
| **测试用例** | TC-008 |

**复现步骤:**
1. 打开聊天界面
2. 不输入任何内容点击发送，或仅输入空格
3. 观察消息列表

**预期结果:**
- 发送按钮应禁用
- 空消息不应被添加到消息列表

**实际结果:**
- 空消息被添加到消息列表
- 可能导致不必要的Agent响应

**影响范围:**
- 用户体验受损
- 可能触发无意义的Agent回复

**修复建议:**
```typescript
// ChatPanel.tsx 第34-48行

  e.preventDefault();
  // 添加更严格的校验
  const trimmedInput = input.trim();
  if (!trimmedInput && uploadedFiles.length === 0) return;
  
  // ... rest of the code
};
```

#### Bug-002: 流程触发依赖顺序 [HIGH] 🟠

| 属性 | 内容 |
|-----|------|
| **ID** | Bug-002 |
| **标题** | 企业画像/客户画像卡片触发依赖特定顺序 |
| **级别** | 🟠 High |
| **模块** | useChatState |
| **测试用例** | TC-002, TC-003 |

**复现步骤:**
1. 直接输入"美国"
2. 不经过"户外"关键词直接输入"万"
3. 观察是否触发相应卡片

**预期结果:**
- 应能独立触发企业画像卡片
- 应能独立触发客户画像卡片

**实际结果:**
- 卡片触发依赖flowIndex顺序
- 必须先触发前一个流程才能触发下一个

**影响范围:**
- 用户无法跳过步骤
- 流程不够灵活

**修复建议:**
```typescript
// useChatState.ts
// 建议添加独立意图识别，而非仅依赖顺序触发

const detectIntent = (text: string): Intent => {
  if (text.includes('美国') || text.includes('欧洲')) {
    return { type: 'COMPANY_PROFILE', data: extractMarket(text) };
  }
  if (text.includes('万') || text.includes('采购')) {
    return { type: 'CUSTOMER_PERSONA', data: extractVolume(text) };
  }
  // ... more intents
};
```

## 5. 改进建议

### 5.1 架构层面

1. **意图识别系统**: 当前基于关键词匹配，建议引入NLP意图识别
2. **状态管理**: 使用Redux或Zustand替代useState，更好地管理复杂状态
3. **API层**: 添加真实API调用，替换Mock数据

### 5.2 体验层面

1. **加载状态**: 添加骨架屏和加载动画
2. **错误处理**: 添加全局错误边界
3. **响应式优化**: 移动端适配

### 5.3 测试层面

1. **E2E测试**: 使用Playwright添加端到端测试
2. **覆盖率**: 目标80%以上代码覆盖率
3. **CI/CD**: 集成自动化测试到部署流程
